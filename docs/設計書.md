# Markdown Quick Memo 設計書

## 構成

| モジュール | 責務 |
| --- | --- |
| `markdown_quick_memo.main` | 引数解析とGUI起動 |
| `markdown_quick_memo.app` | 画面、ショートカット、文書状態、検索、画像プレビュー |
| `markdown_quick_memo.document` | UTF-8読み込み、拡張子補完、アトミック保存 |
| `markdown_quick_memo.hotkey_launcher` | Windowsネイティブホットキーの登録、既存ウィンドウの前面化、アプリ起動 |
| `markdown_quick_memo.markdown_styler` | 元データを変更しない装飾範囲の解析 |
| `markdown_quick_memo.math_renderer` | MathText互換変換、通常・複合数式のPNG描画、先読みとキャッシュ |

## Windowsホットキー起動

`.lnk`のHotkeyプロパティは使用せず、スタートアップから軽量ランチャーを常駐させる。ランチャーはWindows APIの `RegisterHotKey` で `Ctrl + Alt + M` を登録し、Explorerが既存ウィンドウへ問い合わせる際のタイムアウトを回避する。

ホットキー押下時にアプリのウィンドウがあれば前面へ移動し、なければ `MarkdownQuickMemo.exe` を直接起動する。ランチャーは名前付きMutexで多重起動を防ぎ、キーの長押しと短時間の連続入力によるアプリの重複起動を抑止する。

## 編集モデル

`tkinter.Text` 内の文字列をMarkdownの正本とする。解析結果はTextタグとして適用し、保存時にはTextの文字列だけを書き出す。Markdown記号の非表示には `elide` タグを使い、カーソル行は記号を再表示する。

この方式により、見た目を装飾しながら、カーソル位置と保存データの対応を維持する。完全なWYSIWYG編集への変換は行わない。

## ファイル処理

- 読み込み: `utf-8-sig` とし、UTF-8 BOMの有無を許容する。
- 保存: UTF-8（BOMなし）とする。
- 拡張子: 未指定時のみ `.md` を付ける。
- 保存安全性: 同じフォルダへ一時ファイルを書き、成功後に置換する。
- 未保存変更: 新規、読み込み、終了の前に保存確認を表示する。

## Markdown表示

見出し、太字、斜体、太字斜体、取り消し線、リスト、入れ子リスト、チェックボックス、引用、インラインコード、コードブロック、水平線、リンク、画像参照、表、エスケープを解析する。コードブロック内はインライン解析から除外する。

リストの点・番号は `list_marker`、引用記号は `quote_marker` として非表示対象から除外する。コードブロックの開始フェンスと言語指定を別範囲として解析し、フェンスを隠しても言語名は表示する。

水平線と表は `tkinter.Text` の埋め込みウィンドウとして描画する。表はヘッダーとデータ行の間へ水平セパレーターだけを置き、縦線、最上部、最下部の線は描画しない。カーソルが対象ブロック内にある場合は埋め込み表示を破棄し、Markdown原文を表示する。埋め込みウィンドウは `Text.get()` の保存文字列に含まれないため、保存内容は変化しない。

数式はインラインの `$...$` と独立表示の `$$...$$` を解析する。カーソル行以外では `matplotlib.mathtext` によりLaTeX数式をPNGへ変換し、Textの埋め込みウィンドウとして表示する。通常文、見出し、リスト、引用、表セルへ適用し、見出し内では見出しレベルの文字サイズへ連動する。インラインコード、コードブロック、リンク先URL、画像パスは数式解析から除外する。

MathTextが直接扱わない `\tfrac` はコマンド単位で `\frac` へ変換する。変換は描画時だけ行い、Markdown原文には反映しない。通常数式は十分な安全余白を持つキャンバスへ描画し、実際の非背景ピクセル境界を基準に四辺の表示余白を切り揃えて欠損を防ぐ。独立数式では `matrix`、`pmatrix`、`bmatrix`、`Bmatrix`、`vmatrix`、`Vmatrix`、`cases`、`aligned`、`align`、`align*` を解析する。トップレベルの `\\` を行区切り、`&` を列・揃え位置としてセルへ分割し、各セルをMathTextで描画してから数式のベースラインを揃え、括弧類とともに一枚のPNGへ合成する。環境の前後にある通常数式は別画像として描画し、複合環境の高さに対して中央へ配置する。複合数式は最大20行・20列かつ一つの独立数式につき一環境とし、インラインで検出した場合は原文表示へフォールバックする。

MathTextのimport、代表的な通常数式、行列の初期描画はウィンドウ表示後にバックグラウンドで先読みする。通常数式のセル画像は最大256件、複合数式の完成画像は最大64件キャッシュする。MatplotlibとPillowによる画像生成は描画ロック内、Tk画像の生成はメインスレッドで行う。数式解析または画像生成に失敗した場合は、区切り記号を含む数式原文を専用フォントで表示して編集内容を失わないようにする。

リストはインデント幅から階層を判定する。同一階層で連続する番号付き項目は最初の番号から連番表示し、元の番号文字列は変更しない。箇条書きは深さ $0$ を黒丸、それより深い階層を白丸とする。階層ごとにリスト種別と番号カウンターを保持し、番号付き・箇条書きが混在するネストを扱う。

画像は元テキストへ埋め込みオブジェクトを挿入せず、画像記法を `Ctrl + クリック` したときに別ウィンドウで表示する。これにより保存内容と編集位置を変化させない。

ウィンドウの半透明表示は `Ctrl + Shift + O` または編集メニューで切り替える。通常は不透明度100%、半透明時は60%とし、状態をステータス欄へ表示する。透過状態は保存せず、次回起動時は100%へ戻す。

## エラー処理

読み込み、保存、画像表示の失敗はダイアログで通知し、編集中の内容を保持する。認証情報、データベース、ネットワーク保存は扱わない。

## テスト方針

ファイル処理の往復、拡張子補完、BOM読み込み、主要Markdown装飾、コードブロックと言語名、リストの連番・混在ネスト、数式、引用マーカー、水平線、表、リンク・画像参照、ホットキー文字列の解析を単体テストする。数式は互換コマンド、全対応環境、行列上限、インラインでの複合数式拒否を個別に確認する。GUIはTk生成、数式画像、見出しサイズ連動、表セル数式、リストマーカー、埋め込み描画、原文不変性、再描画、破棄のスモークテストを実施する。
