# Markdown Quick Memo 設計書

## 構成

| モジュール | 責務 |
| --- | --- |
| `markdown_quick_memo.main` | 引数解析とGUI起動 |
| `markdown_quick_memo.app` | 画面、ショートカット、文書状態、検索、画像プレビュー |
| `markdown_quick_memo.document` | UTF-8読み込み、拡張子補完、アトミック保存 |
| `markdown_quick_memo.hotkey_launcher` | Windowsネイティブホットキーの登録、既存ウィンドウの前面化、アプリ起動 |
| `markdown_quick_memo.markdown_styler` | 元データを変更しない装飾範囲の解析 |
| `markdown_quick_memo.math_renderer` | MathText互換変換、通常・複合数式のPNG描画、先読みとキャッシュ |
| `markdown_quick_memo.pdf_exporter` | Markdown構造の変換、フォント・数式・画像の配置、PDFのアトミック書き出し |

## Windowsホットキー起動

`.lnk`のHotkeyプロパティは使用せず、現在のユーザーのログオンタスクから軽量ランチャーを常駐させる。タスクは遅延なし、優先度4、同時実行抑止、失敗時再試行を設定し、タスク登録を利用できない場合だけレジストリの `Run` へフォールバックする。ランチャーはWindows APIの `RegisterHotKey` で `Ctrl + Alt + M` を登録し、Explorerが既存ウィンドウへ問い合わせる際のタイムアウトを回避する。

ランチャーはホットキー登録を完了してから `MarkdownQuickMemo.exe --background` を起動し、Tkウィンドウを非表示のまま初期化する。ホットキー押下時は表示・非表示を問わず既存ウィンドウを探し、通常表示へ戻して前面化する。通常の閉じる操作では編集内容を保持したままウィンドウを非表示へ戻し、`Ctrl + Q` またはファイルメニューの「閉じる」で保存確認後に完全終了する。待機プロセスが終了している場合はバックグラウンドモードで再起動し、ウィンドウ生成を最大2秒待ってから表示する。ランチャーは名前付きMutexで多重起動を防ぎ、キーの長押しと短時間の連続入力によるアプリの重複起動を抑止する。

## 編集モデル

`tkinter.Text` 内の文字列をMarkdownの正本とする。解析結果はTextタグとして適用し、保存時にはTextの文字列だけを書き出す。Markdown記号の非表示には `elide` タグを使い、カーソル行は記号を再表示する。

この方式により、見た目を装飾しながら、カーソル位置と保存データの対応を維持する。完全なWYSIWYG編集への変換は行わない。

全文解析では、コード、数式、リンク等の除外範囲をマージ済み区間索引として保持し、二分探索で重複を判定する。解析結果の文字オフセットは、全文から一度生成した行開始位置表によってTkの `行.列` へ変換する。Textタグは種類ごとにまとめて追加し、フォント種別はMarkdownスタイルが変化する区間だけ再判定する。入力直後は文書統計を再計算せず、140ミリ秒の遅延描画時に全文解析・文字数・単語数更新をまとめる。カーソル移動時の位置取得には `Text.count` を使用し、現在行タグは既存範囲だけを解除する。

## ファイル処理

- 読み込み: `utf-8-sig` とし、UTF-8 BOMの有無を許容する。
- 保存: UTF-8（BOMなし）とする。
- 拡張子: 未指定時のみ `.md` を付ける。
- 保存安全性: 同じフォルダへ一時ファイルを書き、成功後に置換する。
- 未保存変更: 新規、読み込み、終了の前に保存確認を表示する。
- PDF書き出し: `Ctrl + Shift + P` またはファイルメニューから任意で実行し、保存済みMarkdownと同じフォルダへ同名の `.pdf` を生成する。未保存または変更中の場合はMarkdownを先に保存する。

PDFはMarkdownを正本とする派生出力であり、PDF自体は編集対象にしない。段落、表、画像などはPython-Markdownの解析結果をReportLabへ変換する。リストの深さ・連番・種類と数式の配置・文字サイズは編集画面と同じ解析結果を直接使用し、数式画像には既存のMathText描画を再利用する。BIZ UDGothic、Segoe UI、Computer Modernの構成を可能な範囲で維持し、Segoe UIを利用できない場合は同梱Robotoへフォールバックする。PDF生成モジュールは操作時だけ遅延読み込みする。相対画像パスはMarkdownの保存フォルダを基準に解決し、外部画像URLは取得しない。既存PDFは確認後に上書きし、一時PDFの生成に成功した場合だけ置換する。

## Markdown表示

見出し、太字、斜体、太字斜体、取り消し線、リスト、入れ子リスト、チェックボックス、引用、インラインコード、コードブロック、水平線、リンク、画像参照、表、エスケープを解析する。コードブロック内はインライン解析から除外する。

リストの点・番号は `list_marker`、引用記号は `quote_marker` として非表示対象から除外する。コードブロックの開始フェンスと言語指定を別範囲として解析し、フェンスを隠しても言語名は表示する。

水平線と表は `tkinter.Text` の埋め込みウィンドウとして描画する。表はヘッダーとデータ行の間へ水平セパレーターだけを置き、縦線、最上部、最下部の線は描画しない。カーソルが対象ブロック内にある場合は埋め込み表示を破棄し、Markdown原文を表示する。埋め込みウィンドウは `Text.get()` の保存文字列に含まれないため、保存内容は変化しない。

数式はインラインの `$...$` と独立表示の `$$...$$` を解析する。カーソル行以外では `matplotlib.mathtext` によりLaTeX数式をPNGへ変換し、Textの埋め込みウィンドウとして表示する。通常文、見出し、リスト、引用、表セルへ適用し、見出し内では見出しレベルの文字サイズへ連動する。インラインコード、コードブロック、リンク先URL、画像パスは数式解析から除外する。

MathTextが直接扱わない `\tfrac` はコマンド単位で `\frac` へ変換する。変換は描画時だけ行い、Markdown原文には反映しない。通常数式は十分な安全余白を持つキャンバスへ描画し、実際の非背景ピクセル境界を基準に四辺の表示余白を切り揃えて欠損を防ぐ。独立数式では `matrix`、`pmatrix`、`bmatrix`、`Bmatrix`、`vmatrix`、`Vmatrix`、`cases`、`aligned`、`align`、`align*` を解析する。トップレベルの `\\` を行区切り、`&` を列・揃え位置としてセルへ分割し、各セルをMathTextで描画してから数式のベースラインを揃え、括弧類とともに一枚のPNGへ合成する。環境の前後にある通常数式は別画像として描画し、複合環境の高さに対して中央へ配置する。複合数式は最大20行・20列かつ一つの独立数式につき一環境とし、インラインで検出した場合は原文表示へフォールバックする。

MathTextのimport、代表的な通常数式、行列の初期描画はウィンドウ表示後にバックグラウンドで先読みする。通常数式のセル画像は最大256件、複合数式の完成画像は最大64件キャッシュする。MatplotlibとPillowによる画像生成は描画ロック内、Tk画像の生成はメインスレッドで行う。数式解析または画像生成に失敗した場合は、区切り記号を含む数式原文を専用フォントで表示して編集内容を失わないようにする。

リストはインデント幅から階層を判定する。同一階層で連続する番号付き項目は最初の番号から連番表示し、元の番号文字列は変更しない。箇条書きは深さ $0$ を黒丸、それより深い階層を白丸とする。階層ごとにリスト種別と番号カウンターを保持し、番号付き・箇条書きが混在するネストを扱う。

## Markdown入力支援

入力支援は`tkinter.Text`のキーイベントとして実装し、装飾表示とは分離してMarkdown原文へ補完文字列を挿入する。補完による挿入・削除は1回の編集操作としてUndo/Redoできる。選択範囲がある場合は、Enterでは選択範囲を改行へ置換し、対記号入力では選択範囲を開き記号と閉じ記号で囲む。

### リストと引用の継続

- リスト行でEnterを押すと、行頭インデント、箇条書き記号、番号付き記号を次行へ継続する。番号付きリストは現在の番号へ1を加算する。
- チェックリストは同じ箇条書き記号と`[ ]`を次行へ挿入する。空のリスト項目でEnterを押すと、項目記号を削除してリストを終了する。
- 引用行は行頭の引用記号と空白を次行へ継続する。引用内リストでは引用接頭辞とリスト接頭辞を合成して継続する。空の引用行では引用接頭辞を削除して引用を終了する。
- `Tab`は現在行のリスト接頭辞の後ろへ半角スペース2個を追加し、`Shift + Tab`は同位置から最大2個の半角スペース、または1個のタブを削除する。引用内リストにも適用する。

リスト・引用の継続判定は、コードフェンス内では実行しない。これにより、コードブロック中の`-`や`>`をリスト・引用として誤補完しない。

### インデントと改行

コードフェンス内の行でEnterを押すと、現在行の先頭インデントを次行へ引き継ぐ。コードフェンス外でも、通常行の先頭にある空白・タブを次行へ引き継ぐ。`Shift + Enter`は構造接頭辞やインデントを追加せず、単純な改行だけを挿入する。

### 対記号とインラインコード

`(`、`[`、`{`、`"`、`'`、バッククォートの入力時は、対応する閉じ記号を挿入してカーソルをその手前へ置く。カーソル直後が同じ閉じ記号の場合は新しい記号を追加せず、その閉じ記号を乗り越える。直前がエスケープ文字の場合や、英文単語内部の引用符では自動補完しない。

バッククォートは選択範囲をインラインコードとして囲める。空行で入力を開始した場合も対記号を作成し、3個目のバッククォート入力時には次行へ同じ長さの閉じフェンスを生成する。カーソルは開始フェンス直後に残るため、言語名を入力してEnterを押すと開始行と閉じ行の間の本文行へ移動できる。コードフェンス自体の自動生成はこの入力支援に限定し、言語名の候補提示やコード内容の自動補完は行わない。

画像は元テキストへ埋め込みオブジェクトを挿入せず、画像記法を `Ctrl + クリック` したときに別ウィンドウで表示する。これにより保存内容と編集位置を変化させない。

ウィンドウの半透明表示は `Ctrl + Shift + O` または編集メニューで切り替える。通常は不透明度100%、半透明時は60%とし、状態をステータス欄へ表示する。透過状態は保存せず、次回起動時は100%へ戻す。

## エラー処理

読み込み、保存、画像表示、PDF書き出しの失敗はダイアログで通知し、編集中の内容を保持する。認証情報、データベース、ネットワーク保存は扱わない。

## テスト方針

ファイル処理の往復、拡張子補完、BOM読み込み、主要Markdown装飾、コードブロックと言語名、リストの連番・混在ネスト、数式、引用マーカー、水平線、表、リンク・画像参照、ホットキー文字列とバックグラウンド起動コマンドの解析を単体テストする。数式は互換コマンド、全対応環境、行列上限、インラインでの複合数式拒否を個別に確認する。GUIはTk生成、数式画像、見出しサイズ連動、表セル数式、リストマーカー、埋め込み描画、原文不変性、再描画、破棄、待機ウィンドウの非表示と文書保持のスモークテストを実施する。

入力支援については、GUIスモークテストでリスト・番号付きリスト・チェックリスト・引用の継続と終了、引用内リスト、コードフェンス内インデント、`Tab`/`Shift + Tab`、`Shift + Enter`、対記号の選択範囲補完、閉じ記号の乗り越え、3個のバッククォートによる閉じフェンス生成を確認する。
